Duration: 1 Week

Database Design Objectives:

Create AcademicYear and Enrollment models for comprehensive student tracking

Implement proper foreign key relationships to eliminate data redundancy

Data Import Objectives:
Create management command for importing student data from JSON files

Design JSON structure with nested faculty and department information

Implement dry-run mode for validation before actual import

Create sample student data file for testing

Set up automatic academic year generation from student records

Authentication API Objectives:
Matric Number Verification Endpoint: POST /api/university/verify-matric/ - Check if student exists in database

Password Creation Endpoint: POST /api/university/create-password/ - Allow verified students to set password

Login Endpoint: POST /api/university/login/ - Authenticate with matric number + password

Logout Endpoint: POST /api/university/logout/ - End user session

Current User Endpoint: GET /api/university/me/ - Retrieve logged-in student information

Password Change Endpoint: POST /api/university/change-password/ - Update password for authenticated users

Session Check Endpoint: POST /api/university/check-session/ - Validate active session

Security Objectives:
Implement password hashing using Django's built-in make_password

Add login attempt limiting (5 attempts before 30-minute lockout)

Create session-based authentication (no tokens stored client-side)

Implement password strength validation (minimum 8 chars, uppercase, lowercase, numbers)

Separate authentication data from core student information

Add account locking mechanism for brute force protection

Admin Interface Objectives:
Register all models with customized Django admin

Create inline admin for StudentAuth within Student admin

Add search and filter capabilities for all models

Implement read-only fields for sensitive data

Create comprehensive list displays with key information

Testing Objectives:

Test edge cases (invalid matric, already registered, wrong password)
